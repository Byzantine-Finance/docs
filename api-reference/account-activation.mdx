Before an end user can submit passkey-based on-chain transactions with gas sponsorship, their account must be activated.

At a high level:

1. You call the passkey activation payload endpoint to retrieve an authorization body to sign.
2. The end user signs this payload with their passkey.
3. You submit the signed payload so Byzantine can sponsor and broadcast the activation transaction.

For the exact parameters and response format, see the **Transactions with passkey** tag in the OpenAPI-driven **API endpoints** section.
